## 2026-04-03

### Q1
질문: localhost:3000번이 쓰이고 있는 것 같다. 어떤 프로세스가 해당 포트를 잡고 있는지 알려줘.

답변: `netstat -ano | findstr :3000`으로 확인 시 PID 10440의 `node.exe`가 `0.0.0.0:3000`에서 LISTENING 상태였다. 종료는 `Stop-Process -Id 10440` 또는 해당 터미널에서 `Ctrl+C`로 가능하다.

### Q2
질문: 분명 로그인을 하고 addCart API를 호출하는데 왜 authorization에 토큰이 없다는 응답을 API 서버에서 보내지?

답변: `httpClient`가 `createClient()`로 앱 로드 시 한 번만 생성되며, 이때 `headers.Authorization`에 `getToken()` 결과가 고정된다. 로그인 전에 페이지를 연 경우 초기값이 빈 문자열이라 이후 `localStorage`에 토큰이 있어도 axios 기본 헤더가 갱신되지 않는다. 해결: 요청 인터셉터에서 매 요청마다 `getToken()`으로 `Authorization`을 설정한다. (서버가 `Bearer <token>` 형식만 인정한다면 인터셉터에서 `Bearer ${token}`으로 맞춰야 한다.)

### Q3
질문: 3000번 포트를 사용하는 프로세스를 킬하고, dev-react-03에서 `npm run start`를 할 때마다 종료해도 죽지 않고 백에서 도는 것 같다. 원인을 파악해줘.

답변: (1) 포트 점유: 조사 시점에 3000은 `node`가 아니라 Docker Desktop의 `com.docker.backend.exe`와 `wslrelay.exe`(WSL 포트 포워딩)가 LISTENING이었다. 이 경우 `taskkill`로 강제 종료하면 Docker 전체가 불안정해질 수 있으므로, 컨테이너의 호스트 포트 매핑을 끄거나 Docker Desktop을 종료하는 편이 안전하다. React만 쓰려면 프로젝트 루트에 `.env`로 `PORT=3001` 등 다른 포트를 주는 방법도 있다. (2) `npm run start`가 안 죽는 느낌: `react-scripts start`는 webpack-dev-server를 띄우는 Node 프로세스 트리가 생기고, Windows에서 터미널 창만 닫거나(Ctrl+C 없이) 에디터가 부모 프로세스만 끊으면 자식 `node.exe`가 고아로 남는 경우가 있다. 또 터미널을 여러 개 열어 `npm start`를 중복 실행하면 프로세스가 여러 개 생긴다. 정리: 종료는 해당 터미널에서 `Ctrl+C`, 안 되면 작업 관리자에서 해당 `node.exe` 종료, 또는 `Get-NetTCPConnection -LocalPort 3000`으로 PID 확인 후 `Stop-Process -Id <PID> -Force`(단, Docker 점유 시에는 위와 같이 Docker 쪽 조치 우선).
