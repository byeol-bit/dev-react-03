## 2026-04-05

### Q1
질문: 현재 토큰 세션이 만료되었는데도 왜 헤더에 로그인 된 것처럼 로그아웃 버튼이 떠있지?

답변: 헤더의 `isLoggedIn`은 **토큰이 “아직 유효한지”가 아니라 `localStorage`에 `token` 값이 있는지**로만 결정된다. (`authStore` 초기값: `getToken() ? true : false`, 로그인 시 토큰만 저장.) JWT 만료 시각은 서버가 검증할 때까지 클라이언트는 모르므로, **만료 후에도 토큰 문자열이 남아 있으면** 계속 로그인 UI가 나온다. 서버가 401을 줄 때만 `http.ts` 인터셉터가 `removeToken()` 후 로그인으로내므로, 그동안 **보호 API를 호출하지 않았거나** 서버가 만료에 **401이 아닌 코드**를 쓰면 토큰이 지워지지 않는다. 참고: `removeToken()`은 Zustand의 `isLoggedIn`을 직접 false로 바꾸지 않지만, 인터셉터는 `window.location.href = "/login"`으로 전체 이동을 하므로 새로고침 시 `getToken()`이 비어 있으면 다시 비로그인으로 맞춰진다. 반대로 401이 나기 전까지는 “만료됐지만 아직 요청 안 함” 상태로 로그아웃 버튼이 보이는 것이 현재 구조상 자연스러운 동작이다.

### Q2
질문: 현재 토큰 만료 시간을 받고 있나?

답변: **아니요.** 로그인 API 타입은 `ILoginResponse { token: string }`만 있고, `expiresIn`·`expires_at` 같은 필드는 정의·사용하지 않는다. 클라이언트는 JWT 페이로드를 디코딩해 `exp`를 읽는 코드도 없고, `localStorage`에는 토큰 문자열만 저장한다. 만료는 서버가 검증하거나 401로 알려줄 때만 반영된다.
