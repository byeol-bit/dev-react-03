## 2026-04-07

### Q1
'0' 패턴에 대한 대체는 배열이 되어야 합니다. 에러를 고쳐줘.

### A1
원인은 `tsconfig.paths.json`의 `compilerOptions.paths` 형식 오류였습니다. 배열(`["src/*"]`)이 아니라 객체여야 하므로 아래처럼 수정했습니다.
- `paths: { "@/*": ["src/*"] }`
검증은 `npx tsc -p tsconfig.json --showConfig`로 수행했고, 설정이 정상 파싱되는 것을 확인했습니다.

### Q2
크라코를 설치했는데 왜 `Cannot find module 'craco-alias'` 에러가 뜨는가?

### A2
`craco`와 `craco-alias`는 다른 패키지입니다. `craco.config.js`에서 `require("craco-alias")`를 사용하고 있어 `craco-alias`가 별도로 필요합니다.
`npm install -D craco-alias`로 설치 후 `npm run start`에서 더 이상 해당 모듈 누락 에러가 발생하지 않고 개발 서버가 기동되는 것을 확인했습니다.

### Q3
이 호출과 일치하는 오버로드가 없습니다.
오버로드 1/3('(options: DefinedInitialDataOptions<unknown, Error, unknown, readonly unknown[]>, queryClient?: QueryClient | undefined): DefinedUseQueryResult<unknown, Error>')에서 다음 오류가 발생했습니다.
'string[]' 형식의 인수는 'DefinedInitialDataOptions<unknown, Error, unknown, readonly unknown[]>' 형식의 매개 변수에 할당될 수 없습니다.
'queryKey' 속성이 'string[]' 형식에 없지만 'Omit<UseQueryOptions<unknown, Error, unknown, readonly unknown[]>, "queryFn">' 형식에서 필수입니다.
오버로드 2/3('(options: UndefinedInitialDataOptions<unknown, Error, unknown, readonly unknown[]>, queryClient?: QueryClient | undefined): UseQueryResult<unknown, Error>')에서 다음 오류가 발생했습니다.
'string[]' 형식의 인수는 'UndefinedInitialDataOptions<unknown, Error, unknown, readonly unknown[]>' 형식의 매개 변수에 할당될 수 없습니다.

### A3
원인은 `@tanstack/react-query` 최신 버전에서 `useQuery(queryKey, queryFn)` 시그니처 대신 `useQuery({ queryKey, queryFn })` 옵션 객체 형태를 요구하기 때문이었습니다.
`src/hooks/useBooks.ts`에서 아래처럼 호출 방식을 변경해 오버로드 타입 오류를 해결했습니다.
- `useQuery({ queryKey: ["books", location.search], queryFn: () => fetchBooks(...) })`
추가로 같은 파일의 미사용 import(`useEffect`, `useState`, `Book`, `Pagenation`)를 정리해 eslint 경고도 함께 제거했습니다.
